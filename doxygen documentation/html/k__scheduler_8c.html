Detailed Description

Contains The System's process queues and all the supporting functionality to schedule process in the kernel.

This module should not be exposed to user programs.

Links a PCB into a specific priority queue.

Parameters
[in,out]PCBpointer to PCB element to link into the respective process queue.

This function is also used to place the idle process in the idle process queue. This poses a potential risk that processes may be initialized with a "priority" lower than what is allowed, but that will only cause that process to never run.

◆ Schedule()

pcb_t* Schedule ( )

Determines which PCB should run next.

Returns
Pointer to PCB of the next process that should run.

This function does not perform any process switching. It simply iterates through the process queues to find the next available process to run.
