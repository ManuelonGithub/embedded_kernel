Detailed Description

Defines all kernel calls that the user has access to.

This is the module that opens the kernel to the user programs. This includes kernel calls and process creation.

priority_t nice ( priority_t  newPriority)

Requests that the running process' priority to be changed.

Parameters
[in]newPriorityNew priority level for the running process to run. Process can be set to a priority between HIGH_PRIORITY and LOWEST_PRIORITY.
Returns
The value of the running's priority after the call was serviced. If the return value is different than the new priority value, then the change of priority was unsuccessful.

This is a preemptive call. When the call is done the kernel scheduler will re-evaluate which process will run based on their priorities.

Receives a character string from the IO server (generated by the user).

Parameters
[in]strpointer to character string to copy string t.
Returns
Number of bytes successfully received from the IO server.

This isn't a inherent kernel call, it is simply a "wrapper" function to a request kernel call where the transaction's parameter/requirements are taken care of. It is an "expensive" operation though, which has 3 to 4 kernel calls in it in order to get all the parameters (valid message box & process ID). Function is useful though as it takes care of filling out the request data for a valid "get user input" interaction with the IO server.
